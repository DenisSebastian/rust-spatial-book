10  Structs y Enums

Diseñando una Ciudad en Rust

Rust ofrece herramientas poderosas para organizar datos. Los Structs y Enums son fundamentales para escribir código limpio y estructurado.

En este capítulo, exploraremos cómo estos dos conceptos funcionan y se relacionan, utilizando la analogía de una ciudad para facilitar su comprensión.

Para usuarios de R: struct es como un list() con tipos fijos

En R, agrupas datos heterogéneos con list():

# R: named list — tipos dinámicos, sin verificación en compilación
punto <- list(lon = -70.6693, lat = -33.4489, nombre = "Santiago")
punto$lon      # -70.6693
punto$nombre   # "Santiago"

En Rust, un struct hace lo mismo, pero con tipos declarados y verificados en compilación:

// Rust: struct — cada campo tiene un tipo fijo
struct Punto {
    lon: f64,
    lat: f64,
    nombre: String,
}

let p = Punto { lon: -70.6693, lat: -33.4489, nombre: String::from("Santiago") };
p.lon     // -70.6693
p.nombre  // "Santiago"

La diferencia clave: el list() de R acepta cualquier tipo en cualquier campo y puede cambiar en tiempo de ejecución. El struct de Rust tiene una forma fija que el compilador conoce — si intentas guardar un texto en un campo f64, falla en compilación, no en tiempo de ejecución.

10.1 Structs: Los Planos de los Edificios

10.1.1 ¿Qué son los Structs?

Un struct es una estructura de datos que agrupa múltiples valores bajo un solo nombre. Estos valores, conocidos como campos, pueden ser de diferentes tipos. Los structs se utilizan para representar entidades concretas, como objetos del mundo real o elementos lógicos.

10.1.2 Tipos de Structs

Rust ofrece tres tipos principales de structs:

  1. Structs Nombrados: Tienen campos con nombres explícitos.
  2. Structs Tuple: Contienen valores sin nombres específicos.
  3. Structs Unitarios: No contienen campos y sirven como marcadores o identificadores.

Relación con el Código

Los structs son fundamentales para modelar datos complejos. Permiten agrupar atributos relacionados de forma estructurada.

🕋 Analogía: Un Plano Arquitectónico

Piensa en un struct como el plano de un edificio. Este plano describe características como:

  • Nombre del edificio
  • Número de pisos
  • Color

Cada struct es un plano único que define cómo construir un tipo de edificio.

Ejemplo de Structs

// Struct nombrado
struct Edificio {
    nombre: String,
    pisos: u32,
    color: String,
}

// Struct tuple
struct Coordenadas(f64, f64);

// Struct unitario
struct SeñalDeTrafico;

10.2 Enums: Tipos de Edificios en la Ciudad

10.2.1 ¿Qué son los Enums?

Un enum es un tipo de dato que representa un conjunto de variantes. Cada variante puede almacenar datos asociados, lo que permite modelar diferentes estados o tipos dentro de un mismo contexto.

Relación con el Código

Los enums son ideales para manejar situaciones en las que un valor puede tener múltiples formas o estados. Además, permiten incluir datos adicionales en cada variante.

🕋 Analogía: Un Catálogo de Tipos de Edificios

Imagina que el enum es un catálogo de edificios en la ciudad. Cada variante describe un tipo de edificio:

  • Una casa con habitaciones y un jardín.
  • Un hospital con camas y un área de emergencias.
  • Una oficina con empleados y pisos.

Ejemplo de Enums

enum TipoDeEdificio {
    Casa { habitaciones: u32, jardin: bool },
    Hospital { camas: u32, tiene_emergencias: bool },
    Oficina { empleados: u32, pisos: u32 },
}

10.3 Ejemplo Integrado: La Ciudad

Combinemos Structs y Enums para representar partes de nuestra ciudad.

// Representamos un edificio concreto
struct EdificioConcreto {
    nombre: String,
    ubicacion: (f64, f64), // Una tupla simple para coordenadas
    tipo: TipoDeEdificio,  // Usamos el Enum dentro del Struct
}

// Enum para representar el uso específico
enum TipoDeEdificio {
    Residencial,
    Comercial,
    Industrial,
}

fn main() {
    let mi_casa = EdificioConcreto {
        nombre: "Torre A".to_string(),
        ubicacion: (10.5, 20.3),
        tipo: TipoDeEdificio::Residencial,
    };
    
    match mi_casa.tipo {
        TipoDeEdificio::Residencial => println!("Es una zona tranquila."),
        _ => println!("Zona de actividad."),
    }
}

10.4 Resumen

En Rust, los struct agrupan datos heterogéneos con tipos fijos — el equivalente de un list() nombrado en R, pero donde el compilador verifica cada campo. Los enum modelan variantes excluyentes con datos adjuntos, algo sin equivalente directo en R. Juntos son la base para tipos espaciales propios como Punto { x: f64, y: f64 } o Geometria { Point(...), Polygon(...) }. En el siguiente capítulo veremos los traits, que permiten definir comportamiento compartido entre structs distintos — el equivalente de métodos S3/S4 en R.

10.5 Reto Práctico

En R, representas un punto geográfico como una lista nombrada:

punto <- list(lon = -70.6693, lat = -33.4489, nombre = "Santiago")
cat(paste0(punto$nombre, ": (", punto$lon, ", ", punto$lat, ")\n"))

Define un struct Punto en Rust con campos lon: f64, lat: f64 y nombre: String. Crea una instancia con los datos de Santiago e imprímela con println!.

Solución 
struct Punto {
    lon: f64,
    lat: f64,
    nombre: String,
}

fn main() {
    let santiago = Punto {
        lon: -70.6693,
        lat: -33.4489,
        nombre: String::from("Santiago"),
    };

    println!("{}: ({}, {})", santiago.nombre, santiago.lon, santiago.lat);
    // Santiago: (-70.6693, -33.4489)
}

A diferencia de R, el struct tiene un tipo fijo en tiempo de compilación. No puedes agregar campos después de su definición. String::from("...") crea un String poseído — necesario porque el campo nombre del struct debe poseer sus datos.
