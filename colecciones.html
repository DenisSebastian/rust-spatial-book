6  Colecciones Esenciales

En R, las colecciones son flexibles por diseño: c(1, "dos", TRUE) simplemente convierte todo al tipo más general sin quejarse. En Rust, cada colección tiene un propósito preciso y un tipo fijo. Esta rigidez no es un obstáculo — es lo que le permite a Rust garantizar que tu código no tendrá sorpresas en tiempo de ejecución.

En este capítulo aprenderás las tres colecciones que verás en casi todos los capítulos siguientes: String, Vec<T> y HashMap<K, V>.

6.1 String y &str: Las Dos Formas del Texto

En R, el texto es texto. No importa si lo vas a modificar o solo leer — siempre usas character. En Rust existe una distinción importante:

  • String: texto que posees y puedes modificar (vive en el heap)
  • &str: referencia a texto que solo lees (inmutable, eficiente)

Piénsalo así: String es como cuando haces nuevo <- paste(viejo, "extra") — creaste algo nuevo que te pertenece. &str es como leer una columna de un dataframe sin copiarlo — solo la estás observando.

Tip

Regla práctica: las funciones suelen recibir &str (no necesitan poseer el texto) y devolver String cuando producen texto nuevo. Si ves &str en un argumento, Rust simplemente promete no modificar ese texto.

fn main() {
    // String: texto que posees y puedes modificar
    let mut ciudad = String::from("Santiago");
    ciudad.push_str(" de Chile");
    println!("{}", ciudad); // "Santiago de Chile"

    // &str: referencia a texto inmutable
    let pais: &str = "Chile";
    println!("País: {}", pais);

    // Las funciones que solo leen prefieren &str
    let longitud = contar_letras(&ciudad);
    println!("Letras: {}", longitud);
}

fn contar_letras(texto: &str) -> usize {
    texto.len()
}

6.1.1 Operaciones comunes de texto

Operación R Rust (String / &str)
Longitud nchar(x) x.len()
Contiene subcadena grepl("sub", x) x.contains("sub")
Mayúsculas toupper(x) x.to_uppercase()
Minúsculas tolower(x) x.to_lowercase()
Separar strsplit(x, ",") x.split(",")
Quitar espacios trimws(x) x.trim()
Concatenar paste(a, b) format!("{} {}", a, b)
Reemplazar gsub("a", "b", x) x.replace("a", "b") 
fn main() {
    let texto = "  datos espaciales  ";

    println!("{}", texto.trim());              // "datos espaciales"
    println!("{}", texto.trim().to_uppercase()); // "DATOS ESPACIALES"
    println!("{}", texto.contains("espacial")); // true

    // split devuelve un iterador — collect() lo convierte en Vec
    let palabras: Vec<&str> = texto.trim().split(' ').collect();
    println!("{:?}", palabras); // ["datos", "espaciales"]
}
Note

split() en Rust devuelve un iterador, no un vector directamente. Necesitas .collect() para materializarlo. Esto puede sentirse extraño al inicio, pero es lo que hace que las cadenas de transformación en Rust sean eficientes sin asignaciones intermedias — similar en espíritu a las “lazy sequences” de algunos paquetes de R.

6.2 Vec<T>: El Vector de Rust

Vec<T> es la colección más usada en Rust. Es el equivalente de c() o vector() en R, pero con una diferencia crítica: todos los elementos deben ser del mismo tipo.

En R, c(1, "dos", TRUE) funciona sin error porque R convierte todo a character. En Rust, Vec<i32> solo acepta enteros i32. Si intentas mezclar tipos, el compilador lo rechaza — no en tiempo de ejecución, sino antes de que el programa corra.

fn main() {
    // Crear un Vec vacío y agregar elementos
    let mut coordenadas: Vec<f64> = Vec::new();
    coordenadas.push(-33.4489); // latitud Santiago
    coordenadas.push(-70.6693); // longitud Santiago

    // Crear con valores iniciales (macro vec!)
    let altitudes = vec![567.0, 820.5, 1200.3, 340.0];

    println!("Coordenadas: {:?}", coordenadas);
    println!("Primera altitud: {}", altitudes[0]);
    println!("Total registros: {}", altitudes.len());
}
Note

La macro vec![...] es azúcar sintáctico para crear un Vec con valores iniciales — equivale a c(...) en R para la creación, pero el tipo se infiere del primer elemento o debes declararlo explícitamente.

6.2.1 Acceso y modificación

fn main() {
    let mut valores = vec![10, 20, 30, 40, 50];

    // Acceso por índice (base 0, a diferencia de R que es base 1)
    println!("Primer elemento: {}", valores[0]); // 10, no 1

    // Remover el último elemento
    let ultimo = valores.pop(); // devuelve Option<i32>
    println!("Removido: {:?}", ultimo); // Some(50)

    // Verificar si contiene un valor
    println!("Contiene 20: {}", valores.contains(&20)); // true

    // Ordenar
    let mut desordenado = vec![3, 1, 4, 1, 5, 9, 2];
    desordenado.sort();
    println!("{:?}", desordenado); // [1, 1, 2, 3, 4, 5, 9]
}

6.2.2 Operaciones comunes sobre vectores

Operación R Rust Vec<T>
Crear vacío vector("numeric", 0) Vec::new() o vec![]
Crear con valores c(1, 2, 3) vec![1, 2, 3]
Agregar elemento c(v, nuevo_val) v.push(nuevo_val)
Longitud length(v) v.len()
Primer elemento v[1] v[0]
Último elemento tail(v, 1) v.last()
Remover último no directo v.pop()
Contiene valor val %in% v v.contains(&val)
Ordenar sort(v) v.sort()
Filtrar v[v > 5] v.iter().filter(...).collect()
Tip

Los índices en Rust comienzan en 0, no en 1 como en R. Esta diferencia genera errores frecuentes al migrar. Si buscas el elemento en posición i desde el pensamiento en R, usa v[i - 1] en Rust — o mejor, replantea la lógica usando iteradores desde el inicio.

6.3 HashMap<K, V>: Diccionarios

Un HashMap<K, V> es el equivalente de una named list o un setNames(c(), c()) en R. Mapea claves (K) a valores (V), donde ambos tipos deben ser homogéneos.

use std::collections::HashMap;

fn main() {
    // Equivalente R: list(Santiago = 7.1, Valparaiso = 0.9, Concepcion = 0.8)
    let mut poblacion: HashMap<String, f64> = HashMap::new();

    poblacion.insert(String::from("Santiago"), 7.1);
    poblacion.insert(String::from("Valparaiso"), 0.9);
    poblacion.insert(String::from("Concepcion"), 0.8);

    // Leer un valor
    if let Some(p) = poblacion.get("Santiago") {
        println!("Población Santiago: {} millones", p);
    }

    // Verificar existencia
    println!("¿Existe Temuco? {}", poblacion.contains_key("Temuco")); // false
}

6.3.1 Inserción con valor por defecto

Una operación muy común en análisis de datos es contar ocurrencias. El método .entry().or_insert() es el equivalente de if (key %in% names(lista)) lista[[key]] + 1 else 1 en R:

use std::collections::HashMap;

fn main() {
    let regiones = vec!["RM", "V", "VIII", "RM", "V", "RM"];
    let mut conteo: HashMap<&str, i32> = HashMap::new();

    for region in &regiones {
        let entrada = conteo.entry(region).or_insert(0);
        *entrada += 1;
    }

    println!("{:?}", conteo);
    // {"RM": 3, "V": 2, "VIII": 1}
}

6.3.2 Operaciones comunes sobre HashMap

Operación R Rust HashMap<K, V>
Crear vacío list() HashMap::new()
Insertar par lista[["clave"]] <- valor map.insert(key, value)
Leer valor lista[["clave"]] map.get("clave") → Option<&V>
Verificar existencia "clave" %in% names(lista) map.contains_key("clave")
Eliminar entrada lista[["clave"]] <- NULL map.remove("clave")
Iterar pares for (k in names(l)) ... for (k, v) in &map { ... }
Total de entradas length(lista) map.len()
Insertar si no existe patrón manual map.entry(k).or_insert(v)
Note

map.get("clave") no devuelve el valor directamente — devuelve Option<&V>. Esto obliga a manejar el caso donde la clave no existe, en lugar de devolver NULL silenciosamente como R. Es más verboso, pero elimina una clase entera de bugs silenciosos comunes en análisis de datos.

6.5 Reto Práctico

Escribe un programa que tome un vector de nombres de regiones (con repeticiones) y produzca un HashMap con la frecuencia de cada una. Luego imprime las regiones ordenadas por frecuencia de mayor a menor.

Pistas: - Construye el HashMap usando .entry().or_insert(0) - Para ordenar por valor, convierte el HashMap a un Vec de pares con .collect() y luego usa .sort_by() - sort_by(|a, b| b.1.cmp(&a.1)) ordena de mayor a menor por el segundo elemento del par

use std::collections::HashMap;

fn main() {
    let regiones = vec![
        "RM", "V", "VIII", "RM", "IX", "V", "RM",
        "IX", "RM", "V", "VIII", "RM",
    ];

    let mut conteo: HashMap<&str, i32> = HashMap::new();
    for region in &regiones {
        *conteo.entry(region).or_insert(0) += 1;
    }

    // Convertir a Vec para ordenar
    let mut pares: Vec<(&&str, &i32)> = conteo.iter().collect();
    pares.sort_by(|a, b| b.1.cmp(a.1));

    println!("Región      | Frecuencia");
    println!("------------|----------");
    for (region, freq) in pares {
        println!("{:<12}| {}", region, freq);
    }
}

Salida esperada:

Región      | Frecuencia
------------|----------
RM          | 5
V           | 3
VIII        | 2
IX          | 2

6.6 Resumen

Concepto R Rust Diferencia clave
Texto mutable character String Rust distingue posesión vs referencia
Texto inmutable character &str &str es solo una vista, no copia
Secuencia c(), vector() Vec<T> Un solo tipo; índices base 0
Diccionario named list HashMap<K, V> .get() devuelve Option, no NULL
Colección mixta list() struct En Rust los tipos mixtos requieren estructura explícita

Con estas tres colecciones puedes representar la mayoría de las estructuras de datos que usarías en análisis espacial: series de coordenadas (Vec<f64>), índices de localidades (HashMap<String, Vec<f64>>), y texto con nombres de capas o archivos (String/&str). En los capítulos siguientes las verás en acción constantemente.
